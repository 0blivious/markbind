    • Using Variables

    You can insert variables to be replaced by pre-defined values anywhere in your site. MarkBind treats any text surrounded with double curly braces as a variable, e.g. {{variable_name}}

    Some helpful variables are provided by MarkBind for your convenience, as shown below.

    Specifying Path Reference

    Use {{baseUrl}} for absolute path reference of images and resource files so that the path could be handled correctly when deployed. The {{baseUrl}} is parsed from the project root (where site.json located).

    Font Awesome icons

    This asset is provided by Font Awesome under their free license.

    Traditional usage:

    <span class='fas fa-chess-knight'></span>

    MarkBind usage:
    {{fas_chess_knight}}

    Font Awesome has three different styles for their free icons, each with their own prefix: Solid (fas), Regular (far) and Brands (fab).
    Be sure to check Font Awesome's full list of icons and which styles each icon supports.

    MarkBind offers a shorter syntax by transforming the icon's name. It is a two-step process:

    1. Shift the icon's prefix to replace fa, the icon's name first two characters. e.g fas fa-chess-knight becomes fas-chess-knight
    2. Next, replace all instances of - with _ in the icon's name. e.g. fas-chess-knight becomes fas_chess_knight

    After doing so, you are able to use the icons anywhere on your site by surrounding an icon's shortened name with double curly braces.
    e.g. {{fas_fighter_jet}} will be rendered as

    Glyphicons

    This asset is provided by Glyphicons via Bootstrap 3.

    Traditional usage:

    <span class='glyphicon glyphicon-hand-right' aria-hidden='true'></span>

    MarkBind usage:
    {{glyphicon_hand_right}}

    Here is the list of provided glyphs from Bootstrap.

    MarkBind offers an alternate syntax by transforming the icon's name.
    It is done by replacing all instances of - with _ in the glyph's name. e.g. {{glyphicon-hand-right}} becomes {{glyphicon_hand_right}}

    After doing so, you are able to use the glyphs anywhere on your site by surrounding a glyph's modified name with double curly braces.
    e.g. {{glyphicon_hand_right}} will be rendered as

    Timestamp

    Use {{timestamp}} to insert a UTC time snippet that indicates when the page was generated.

    {{timestamp}} will be rendered as: Tue, 06 Nov 2018 11:17:42 GMT

    MarkBind Link

    Reference or support MarkBind by using {{MarkBind}} to insert a link to our website.

    {{MarkBind}} will be rendered as: MarkBind 1.14.1

    Add your own variables

    In _markbind/variables.md, you can define your own variables that can be used anywhere in your site. The format is as follows.

    <span id="year">2018</span>

<span id="options">
* yes
* no
* maybe
</span>

    Each variable must have an id and the value can be any MarkBind-compliant code fragment. The id should not contain - and .. For example, search-option and search.options are not allowed.

    In any file in your site, you can include the variable by surrounding the variable's id with double curly braces, e.g. {{options}}

    Variables can refer to other variables that are declared earlier, including helpful variables provided by MarkBind.

    This is allowed, MarkBind helpful variables can be used.

    <span id="right_hand">{{glyphicon_hand_right}}</span>

    This is also allowed, the second variable will be assigned the contents of the first variable.

    <span id="first">This is the first variable.</span>
    <span id="second">{{first}}</span>
    <span id="third">{{second}}</span>

    This is forbidden, as the fourth variable is not declared yet, and will not be rendered correctly.

    <span id="third">{{fourth}}</span>
    <span id="fourth">This is the fourth variable.</span>

    Note: If the variable being referenced contains HTML tags, MarkBind may escape the tags and render it literally. For example:

    If we declare the variables as follow:

    <span id="right_hand">{{glyphicon_hand_right}}</span>
    <span id="right_hand_2">{{right_hand}}</span>

    The following will be rendered:


    <span class="glyphicon glyphicon-hand-right" aria-hidden="true">

    You must use the safe filter when using such variables:

    If we use the safe filter for the second variable:

    <span id="right_hand">{{glyphicon_hand_right}}</span>
    <span id="right_hand_2">{{right_hand | safe}}</span>

    The following will be rendered:
